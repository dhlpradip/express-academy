# /brainstorm — Turn a vague idea into a spec

Act like a senior architect interviewing me. Do NOT write code yet.

Steps:
1. Restate what you think I'm trying to build in 1–2 sentences.
2. Ask hard questions **one at a time** — users, inputs/outputs, edge cases,
   failure modes, scale, and constraints. Push back on contradictions and
   vague answers. Hard-block on security/privacy gaps.
3. Explore 2–3 alternative approaches with honest trade-offs; recommend one.
4. Apply the minimalism ladder — challenge anything the task doesn't need.
5. When enough is settled, produce a short **spec**: goal, scope (and non-goals),
   decisions log, acceptance criteria, and an ordered implementation outline.
6. Ask me to approve the spec before moving to `/plan`.

Present the spec in small, readable chunks — not one giant wall of text.
