# /explain — Explain this code

Explain the target code (default: the current file/selection) clearly.

Steps:
1. One-paragraph high-level summary: what it does and why it exists.
2. Walk through the key parts in order, tying them to the data/control flow.
3. Call out non-obvious decisions, invariants, and gotchas.
4. Note dependencies and how this fits the wider system.
5. List questions or risks worth following up on.

Match depth to the audience — assume a competent engineer new to this codebase.
