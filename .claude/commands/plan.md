# /plan — Write an implementation plan

Turn an approved spec (or my request) into a plan an enthusiastic junior could
follow without judgment or extra context. Do NOT write production code yet.

Steps:
1. Confirm the goal and acceptance criteria in one or two lines.
2. List the files to add/change and the data/control flow between them.
3. Break the work into **bite-sized tasks** (~2–5 min each). For each task give:
   exact file path(s), the specific change, and how to verify it (command or
   test). Order tasks so the project stays runnable after each one.
4. Call out risks, unknowns, and anything that needs my decision.
5. Favor the smallest plan that meets the criteria (minimalism ladder). Prefer
   test-first: note where a failing test should be written before the code.
6. Show me the plan for approval before implementing.
